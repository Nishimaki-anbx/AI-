# タスク管理エージェント MVP 設計メモ

## 目的

自分用のタスク管理エージェントを、小さな CLI ツールとして作り始める。
最初はローカル保存を前提にし、後から AI 連携を追加できる形にする。

## 初期方針

- 操作方法は CLI から始める
- 保存形式は JSON にする
- タスク情報はローカルファイルに保存する
- AI 連携は初期 MVP には含めず、後続の設計で追加する
- 設計判断は都度 `docs/design/` に残す

## JSON を選ぶ理由

- 実装と確認が簡単
- Git の差分で変更内容を追いやすい
- 初期 MVP では複雑な検索や集計より、データ構造の試行錯誤を優先したい
- 将来 SQLite などへ移行する場合も、変換元として扱いやすい

## 初期データ案

```json
{
  "tasks": [
    {
      "id": "task-001",
      "title": "README にセットアップ手順を追加する",
      "status": "todo",
      "priority": "medium",
      "due_date": null,
      "notes": [],
      "created_at": "2026-04-27T00:00:00+09:00",
      "updated_at": "2026-04-27T00:00:00+09:00"
    }
  ]
}
```

## 初期ステータス案

- `todo`: 未着手
- `doing`: 進行中
- `done`: 完了

## 将来の拡張メモ

- 会議要約テキストから関連タスクの追加・更新案を作る
- 日次対応メモから進捗、完了、次アクションを抽出する
- AI が提案した変更は、反映前に人間が確認できるようにする
- JSON の構造変更が必要になったら、移行手順も設計資料に残す
